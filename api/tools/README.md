# API Endpoint Generator

A minimal tool to generate FastAPI endpoints from OpenAPI specifications.

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install uv
uv lock
uv sync
```

## Usage

```bash
python generate_endpoints.py <openapi_spec.yaml> <output_file.py> --service-name <service_name>
```

### Options

- `--no-schemas`: Skip automatic schema generation (schemas will be generated by default)

## What it generates

1. **Pydantic schemas** (automatic): Generates data models in `src/schemas/{service_name}.py` using datamodel-codegen
2. **FastAPI endpoints**: Generates endpoints in the specified output file

### Example

```bash
python generate_endpoints.py ../public/<my service>.yaml ../../services/<my service>/src/api/v1/endpoints.py --service-name <my service>
```

## Features

- **Auto-generates Pydantic schemas** from OpenAPI specs using datamodel-codegen
- Generates idempotent FastAPI endpoints with proper dependency injection
- Converts camelCase path parameters to snake_case
- Includes proper type annotations for service dependencies
- Handles CRUD operations (GET, POST, PUT, DELETE)
- Automatically handles List[Model] response types for array responses
- Generates clean, formatted code

## Generated Code Structure

- Uses `Depends()` for service layer injection
- Properly typed service parameters
- Converts OpenAPI paths to FastAPI routes
- Includes error handling and HTTP status codes
- Imports required schemas and service classes

## Requirements

The generated code expects:
- Schemas to exist in `src/schemas/{service_name}.py`
- Service layer to exist in `src/services/{service_name}_service.py`
- Service class named `{ServiceName}Service`
- Dependency function named `get_{service_name}_service`
